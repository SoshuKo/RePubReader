2. 「残り作業」補足（見落とし防止）
==================================================

あなたが列挙した4項目で本筋は合っている。
ただし以下は実装時に必ず並行確認すること。

- 切断/再接続時の同期復元
- host交代しない前提でも「host切断時のルーム挙動」
- command実行時の権限漏れ
- モバイルSafariでの操作イベント競合（長押し/タップ）


==================================================
3. Python(VPS)で手動設定が必要な項目（ConoHa含む）
==================================================

A. 必須環境変数（.env）
- REPUNIS_HOST_ADMIN_PASSWORD=<ホスト管理パス>
- REPUNIS_ALLOWED_ORIGINS=<GitHub PagesのURLをカンマ区切り>
  例: https://soshuko.github.io
- REPUNIS_MAX_ROOMS=<同時ルーム上限>
- REPUNIS_ROOM_TTL_SEC=<無通信ルームの自動破棄秒>

B. ConoHa VPS 側
- Python 3.11+ を導入
- systemd サービス作成（自動起動）
- UFW/ファイアウォールで使用ポート許可（例 8000）
- Nginx リバースプロキシ設定
  - /ws を websocket upgrade 付きで転送
  - HTTPS終端（Let's Encrypt）
- ドメインまたは固定URLを決める

C. CORS / セキュリティ
- GitHub Pagesドメインのみ許可
- host専用APIは admin_password 必須
- ルームパス4桁はサーバー照合
- ログに平文パスワードを出さない

D. 運用手順（手動）
- デプロイ後に /api/rooms が空配列で200を返すか確認
- host作成 -> guest参加 -> WS接続維持を確認
- invite_battle -> accept_battle -> phase=battle の順で受信確認
- 異常時は serverログの session_token と type で追跡

E. トラブル時チェック
- WebSocketが101になっているか
- mixed content（httpsページ -> ws://）になっていないか
- CORS preflight(OPTIONS)が通っているか
- systemd再起動後に環境変数が読めているか
